# 🎸 Vintage Guitar Market Intelligence

> A multi-modal machine learning pipeline that predicts 6-month price trajectories for vintage electric guitars (1950s–1980s) in the North American secondary market.

Built with **Python**, **PyTorch**, **DistilBERT**, and **Plotly Dash**.

---

## Architecture

```
Listing Description (text)
        │
  [DistilBERT Encoder]  ──→  128-dim text embedding
        │                              │
        │                          [Concat]
        │                              │
Structured Features (tabular)     [Fusion MLP]
        │                              │
  [Tabular MLP]  ──────→  64-dim   [Mean + Log-Var]
                                       │
                            Predicted Price + Uncertainty (95% CI)
```

**Key design decisions from the research proposal:**
- **Multi-modal fusion**: Integrates unstructured text with structured tabular data, capturing subtle signals buried in descriptions (e.g. "OHSC", "refretted", "non-original")
- **Uncertainty quantification**: Gaussian NLL loss trains the model to output calibrated confidence intervals — displayed as shading on the dashboard
- **Outlier removal**: Z-score filtering per brand/year group removes celebrity-sale price spikes
- **Player Grade cluster**: Treated as a separate condition category, not just a lower-tier "Good"

## Running the Pipeline

### Step 1 — Scrape listings
```bash
python scraper.py
```
Outputs `data/raw_listings.csv`. Scrapes sold listings for Fender, Gibson, Gretsch, Rickenbacker, and Epiphone across the 1950s–1980s decade filters on Reverb.

> ⚠️ Reverb's HTML structure changes occasionally. If scraping breaks, inspect the listing card selectors in `scraper.py` and update `parse_listing_card()`.

### Step 2 — Preprocess
```bash
python preprocess.py
```
Outputs `data/processed_listings.csv` with:
- Originality score (-1 / 0 / 1)
- Player Grade flag
- Normalized condition labels
- Pickup configuration (SSS, HH, etc.)
- Outliers removed
- Prices standardized to USD

### Step 3 — Train
```bash
python train.py
```
Trains for 20 epochs with cosine LR scheduling. Saves `checkpoints/best_model.pt` (best validation loss). Logs train loss, val loss, and val MAE in dollars per epoch.

Recommended: run on a GPU. CPU training is possible but slow due to DistilBERT.

### Step 4 — Launch dashboard
```bash
python dashboard.py
```
Open [http://localhost:8050](http://localhost:8050)

**Dashboard features:**
- Filter by brand
- Toggle USD / CAD
- Historical sold price chart (yearly averages)
- AI 6-month forecast with 95% confidence interval shading
- Buy / Hold / Sell signal badge
- Data freshness timestamp

> If `checkpoints/best_model.pt` doesn't exist yet, the dashboard runs in **demo mode** with extrapolated prices — useful for development before training completes.
